现在的课题是**视觉到触觉的低时延预测与缓存**。背景是 XR、机器人远程操作这类场景里，触觉反馈对时延要求非常高，但如果每次都等真实接触发生后再生成或传输触觉反馈，时间肯定来不及。所以思路是：**在接触发生前，根据视觉信息提前预测未来可能产生的触觉反馈**。

我的研究是在这个基础上继续改进。我们不想一上来就用很重的生成模型直接生成整张触觉图像，而是先利用已有的数据集（包含视频里的每一帧和每一帧对应的机械臂末端传感器的触觉图像），将对应的视觉和触觉缓存下来，也就是机械臂即将触碰的物体的局部区域和对应的触觉图像缓存下来。在线部分我们只需要预测出机械臂的接触区域，加上运动特征去检索缓存库里的最相似的触觉图像，若找不到就还是走生成触觉图像。

目前整体思路是：

1. **用 CoTracker 跟踪机械臂末端**
    - 在视频中跟踪机械臂末端或触觉传感器上的点；
    - 得到机械臂的运动方向和速度。
2. **根据运动方向预测未来接触区域**
    - 沿机械臂运动方向画出可能接触的区域；
    - 或者截取机械臂前方的局部图像；
    - 让轻量网络输出一个“接触热力图”，表示局部区域里哪里最可能被碰到。
3. **提取接触区域的视觉特征**
    - 包括局部形状、材质、纹理；（但是我们的数据集中触觉图像中应该提取不出来材质和纹理的特征）
    - 再结合机械臂速度和方向，估计接触强度或力的大小。
4. **用这些特征去匹配触觉图像**
    - 建立一个视觉—触觉缓存库；
    - 输入是视觉特征，比如接触区域、材质、速度方向；
    - 输出是历史上相似情况下的触觉图像或触觉特征；
    - 这样在线阶段就不用每次重新生成触觉反馈，可以直接快速检索。
