## 1. EgoPAT3Dv2 是怎么做 2D to 3D 的？

先说结论：**EgoPAT3Dv2 并不是像传统几何方法那样，通过相机内参 + 深度图 + 三角测量，严格把 2D 点反投影到 3D。它更像是用神经网络从 RGB 视频中直接回归未来动作目标的 3D 坐标。**

也就是说，它做的是：

RGB video frames→future 3D action target\text{RGB video frames} \rightarrow \text{future 3D action target}RGB video frames→future 3D action target

而不是：

2D point+depth+camera parameters→3D point\text{2D point} + \text{depth} + \text{camera parameters} \rightarrow \text{3D point}2D point+depth+camera parameters→3D point

EgoPAT3Dv2 的目标是从第一视角 RGB 视频中预测人手动作最终会到达的 **3D action target location**。论文强调，它增强后的方法可以只使用 RGB 图像，不再像之前方法那样需要 3D point clouds 和 IMU 输入。

### 它的大致逻辑可以理解成三步

#### 第一步：从 2D 视频中提取动作线索

输入是一段 egocentric RGB video，也就是第一视角视频。模型会观察：

- 手的位置；
- 手的运动轨迹；
- 场景中的物体；
- 人手朝哪个区域移动；
- 目标物体可能在哪里。

这一步本质上是从 2D 图像中学习动作趋势。

#### 第二步：利用大模型和人体先验

EgoPAT3Dv2 改进了原来的 baseline，引入了 large pre-trained model 和 human prior knowledge。它不是单纯靠几何计算，而是利用大规模视觉模型学到的空间、物体和动作先验。

通俗讲就是：  
模型看过很多视觉场景之后，会学到一些隐含规律，比如“手朝杯子移动时，最终目标大概率在杯子表面/把手附近”。

#### 第三步：直接预测 3D 坐标

最后输出的是未来动作目标的 3D coordinate，而不是只输出 2D 框或 2D mask。

所以它的 2D to 3D 本质是：

> **通过数据监督，让模型学习 RGB 视频特征到 3D 目标坐标之间的映射。**

它依赖训练数据中有 3D target 的标注，但推理时可以只用 RGB 视频。
